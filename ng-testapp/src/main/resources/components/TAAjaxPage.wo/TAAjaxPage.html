Ajax functionality

Ajax functionality is still very primitive (as can be seen from the look of ng-js.js).

We do have one feature missing from WO's Ajax.framework - we allow the updating of multiple update containers (from one action/response).
We do this by constructing a multipart response, with each part of the response corresponding to a targeted container.
The approach seems to work fine, but we'll see whether we keep it as-is (or if we'll resort to something more complex). I like the multipart response-approach though, since it's a pretty well established standard method of returning many things in one resposne.

Another thing is that our page cachce is designed from the ground up to handle Ajax requests. Still needs some work to properly handle multi-container updates though.
